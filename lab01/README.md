# Лабораторная работа №1

Порождающий паттерн проектирования **Одиночка** или Singleton.

## Предметная область

Была выбрана следующая предметная область: приложение для управления заметками. Приложение позволяет пользователю создавать, редактировать и удалять текстовые заметки. В данном приложении несколько независимых компонентов нуждаются в записи событий (Logger), что делает применение паттерна Одиночка очевидным и обоснованным.

## Описание проблемы

Приложение состоит из нескольких компонентов, каждый из которых порождает события, требующие логирования:

- NoteRepository — отвечает за сохранение и удаление заметок во внутреннем хранилище.
- NoteEditor — обрабатывает редактирование содержимого заметки.
- NoteListController — управляет списком заметок: поиск, сортировка, навигация.

**Проблема**

- Если каждый компонент создаёт собственный экземпляр Logger (new Logger()), возникает несколько независимых объектов с отдельными списками записей. Хронология событий рассыпается — невозможно восстановить точный порядок действий пользователя.

## Применение паттерна Одиночка

Паттерн Одиночка гарантирует, что класс Logger будет иметь единственный экземпляр на всё приложение, и предоставляет к нему глобальную точку доступа.
